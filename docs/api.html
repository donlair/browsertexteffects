API Reference

Complete reference for the shared runtime exports from browsertexteffects and the per-effect modules under browsertexteffects/effects/*.

createEffectWith #

function createEffectWith(
  container: HTMLElement,
  text: string,
  effectDefinition: EffectDefinition<TConfig>,
  config?: Partial<TConfig> & { lineHeight?: number; fillContainer?: boolean; extraRows?: number; onComplete?: () => void }
): EffectHandle

Creates an animated text effect inside container using an imported effect descriptor. Returns an EffectHandle — call start() on it to begin the animation.

Parameter Type Description
container HTMLElement The DOM element that will contain the animated text. BrowserTextEffects writes into this element and manages its contents during the animation.
text string The text to animate. Multi-line text (with \n) is fully supported.
effectDefinition EffectDefinition<TConfig> An effect descriptor imported from a module such as browsertexteffects/effects/beams.
config object Optional config overrides for the imported effect. Only provide the properties you want to change — everything else falls back to that effect's defaults. Also accepts lineHeight, fillContainer, extraRows, and onComplete.
Returns EffectHandle 
import { createEffectWith } from "browsertexteffects";
import { beamsEffect } from "browsertexteffects/effects/beams";

const el = document.getElementById("my-text");
const handle = createEffectWith(el, "Hello, World!", beamsEffect);
handle.start();

// Stop at any time:
handle.stop();
Note: BrowserTextEffects requires a browser environment. It uses requestAnimationFrame and DOM APIs and is not compatible with server-side rendering.

createEffectOnScrollWith #

function createEffectOnScrollWith(
  container: HTMLElement,
  text: string,
  effectDefinition: EffectDefinition<TConfig>,
  config?: Partial<TConfig> & { lineHeight?: number; fillContainer?: boolean; extraRows?: number; onComplete?: () => void }
): EffectHandle

Same signature as createEffectWith, but the animation starts automatically when the container scrolls into the viewport. Uses IntersectionObserver internally with a 10% threshold; the observer disconnects after the first trigger so the animation plays once per page load.

Parameter Type Description
container HTMLElement The element to watch and animate.
text string The text to animate.
effectDefinition EffectDefinition<TConfig> An effect descriptor imported from a tree-shakable effect module.
config object Optional effect config overrides, plus lineHeight, fillContainer, extraRows, and onComplete.
Returns EffectHandle 
import { createEffectOnScrollWith } from "browsertexteffects";
import { decryptEffect } from "browsertexteffects/effects/decrypt";

const el = document.getElementById("my-text");

// Starts automatically when el scrolls into view (10% threshold)
const handle = createEffectOnScrollWith(el, "Scroll triggered!", decryptEffect);

// You can still stop it manually:
handle.stop();

EffectHandle #

interface EffectHandle {
  start(): void;
  stop():  void;
}

Returned by both createEffectWith and createEffectOnScrollWith, as well as the per-effect helper functions. Provides manual control over the animation loop.

Method Returns Description
start() void Begins the animation using requestAnimationFrame. Calling start() again while the animation is already running is a no-op.
stop() void Cancels the animation at the current frame. The text remains in whatever state it was in when stopped.

Color #

interface Color {
  rgbHex: string; // six-character hex, no leading "#"
}

Represents a single RGB color used in effect configurations. Create colors with the color() helper rather than constructing the object directly.

color() helper

function color(hex: string | number): Color

Accepts a hex string or an XTerm-256 integer code (0–255).

Input Type Example
Hex string string color("ff0000") or color("#ff0000")
XTerm-256 code number color(196) → bright red 
import { color } from "browsertexteffects";
import { createBeamsEffect } from "browsertexteffects/effects/beams";

const handle = createBeamsEffect(el, "Hello!", {
  finalGradientStops: [
    color("#8A008A"),
    color("00D1FF"),
    color(255), // XTerm-256: white
  ],
});

Additional color utilities also exported from browsertexteffects:

Export Signature Description
xtermToHex (xterm: number) => string Converts an XTerm-256 code to a six-character hex string.
rgbInts (c: Color) => [number, number, number] Unpacks a Color into [r, g, b] integers (0–255 each).
adjustBrightness (c: Color, factor: number) => Color Multiplies each RGB channel by factor and clamps to 0–255.
colorPair (fg: Color | null, bg?: Color | null) => ColorPair Creates a foreground/background color pair used by some effect internals.

GradientDirection #

type GradientDirection =
  | "vertical"
  | "horizontal"
  | "radial"
  | "diagonal"

Controls how a gradient is mapped across the canvas. Used in config properties like finalGradientDirection.

Value Description
"vertical" Gradient runs top to bottom.
"horizontal" Gradient runs left to right.
"radial" Gradient radiates outward from the center.
"diagonal" Gradient runs from top-left to bottom-right.

EasingFunction #

type EasingFunction = (progress: number) => number

A function that maps a linear progress value (0–1) to an eased value (0–1). Used in effect configs that accept an easing parameter to control how motion accelerates or decelerates.

BrowserTextEffects exports a full set of standard easing functions via the easing namespace:

import { easing } from "browsertexteffects";
import { createSlideEffect } from "browsertexteffects/effects/slide";

const handle = createSlideEffect(el, "Hello!", {
  easing: easing.outBounce,
});

Available easing functions in the easing namespace include: linear, inSine, outSine, inOutSine, inQuad, outQuad, inOutQuad, inCubic, outCubic, inOutCubic, inQuart, outQuart, inOutQuart, inQuint, outQuint, inOutQuint, inExpo, outExpo, inOutExpo, inCirc, outCirc, inOutCirc, inBack, outBack, inOutBack, inElastic, outElastic, inOutElastic, inBounce, outBounce, inOutBounce.

Custom easing: Any function with the signature (progress: number) => number is a valid EasingFunction. You can pass a custom function directly.

Common Config Patterns #

Many effects share the same config property names for comparable features. These are the most common ones you'll encounter across effect configs.

Property Type Description
finalGradientStops Color[] Color stops for the gradient applied to characters in their final resting state. Accepted by most effects.
finalGradientSteps number Number of interpolated steps between each gradient stop color. Higher values produce smoother gradients.
finalGradientFrames number Number of animation frames over which the final gradient is applied when characters arrive.
finalGradientDirection GradientDirection Direction of the final gradient across the canvas. One of "vertical", "horizontal", "radial", "diagonal".
easing EasingFunction Controls how character movement accelerates/decelerates. Accepts any function from the easing namespace or a custom (t: number) => number.
speed number General speed multiplier for the effect. Interpretation varies by effect (frames per step, distance per frame, etc.).
grouping string Grouping of characters for sequenced effects. Values: "row", "rowBottomToTop", "column", "columnRightToLeft", "diagonal", "diagonalTopRightToBottomLeft", "diagonalTopLeftToBottomRight", "diagonalBottomRightToTopLeft", "centerToOutside", "outsideToCenter".
reverseDirection boolean Reverses the order in which character groups are animated. Available in directional effects like slide, sweep, wipe.
lineHeight number Row height in pixels for the absolute-positioned renderer. Passed at the top level of the config object (not effect-specific). Defaults to the container's computed line-height.
fillContainer boolean When true, pads the text with invisible characters to fill the container's parent dimensions, making effects animate across the full area instead of just the text characters.
extraRows number Adds extra rows of padding above and below the text when fillContainer is enabled. Useful for single-line elements like headings where the parent has minimal height.
onComplete () => void Callback fired once when the animation fully completes. Passed at the top level of the config object.
Tip: The exact properties available for each effect are documented on each effect's page. Click any module in the Effect Modules grid above to see its full config table.

Example combining multiple common patterns:

import { color, easing } from "browsertexteffects";
import { createSweepEffect } from "browsertexteffects/effects/sweep";

const handle = createSweepEffect(el, "Hello, World!", {
  // Final gradient: purple → cyan → white, top to bottom
  finalGradientStops: [color("8A008A"), color("00D1FF"), color("ffffff")],
  finalGradientSteps: 12,
  finalGradientDirection: "vertical",

  // Animate column by column
  grouping: "column",
  reverseDirection: false,

  // Use outCubic movement
  easing: easing.outCubic,

  // Fire a callback when done
  onComplete: () => console.log("Animation complete!"),
});
handle.start();

Default Configs #

Every effect module exports a default*Config object containing its default values. Spread it into your config to get full IDE autocompletion over all available properties:

import { createBeamsEffect, defaultBeamsConfig } from "browsertexteffects/effects/beams";

const handle = createBeamsEffect(el, "Hello!", {
  ...defaultBeamsConfig,   // IDE shows every beams property
  beamDelay: 3,            // override just what you need
});
