Getting Started

Add animated text effects to any web page in minutes. BrowserTextEffects works with any framework — or plain HTML.

Installation #

Install from npm:

npm install browsertexteffects

Then import in your JavaScript or TypeScript:

import { createBeamsEffect } from "browsertexteffects/effects/beams";
Note: BrowserTextEffects requires a browser environment. It uses requestAnimationFrame and DOM APIs and is not compatible with server-side rendering.

Basic Usage #

Import the effect you want, pass a container element and the text to animate, then call start() on the returned handle.

import { createBeamsEffect } from "browsertexteffects/effects/beams";

const el = document.getElementById("my-text");
const handle = createBeamsEffect(el, "Hello, World!");
handle.start();

// Stop at any time:
handle.stop();

The per-effect helper returns an EffectHandle with two methods:

  • start() — begins the animation. Calling again while already running is a no-op.
  • stop() — cancels the animation at the current frame.
Tip: Every effect has its own tree-shakable subpath export under browsertexteffects/effects/*. Browse them all live in the Showroom.

Custom Config #

Every effect helper accepts an optional config object. Pass only the properties you want to override — everything else falls back to that effect's defaults.

import { createSlideEffect } from "browsertexteffects/effects/slide";

const el = document.getElementById("my-text");
const handle = createSlideEffect(el, "Custom Config", {
  grouping: "column",
  reverseDirection: true,
});
handle.start();

For TypeScript autocompletion over all available config properties, import and spread the matching default*Config export:

import {
  createSlideEffect,
  defaultSlideConfig,
} from "browsertexteffects/effects/slide";

const handle = createSlideEffect(el, "Custom Config", {
  ...defaultSlideConfig,   // spreads all defaults — IDE shows every property
  grouping: "column",
  reverseDirection: true,
});
See also: The API Reference documents every config property. Each effect page has a full config table with types, defaults, and descriptions.

Scroll-Triggered Effects #

Use the per-effect scroll helper to automatically start the animation when the container enters the viewport. The effect fires once, when at least 10% of the element becomes visible.

import { createDecryptEffectOnScroll } from "browsertexteffects/effects/decrypt";

const el = document.getElementById("my-text");

// Starts automatically when el scrolls into view
const handle = createDecryptEffectOnScroll(
  el,
  "Scroll triggered!"
);

// You can still stop it manually:
handle.stop();

createDecryptEffectOnScroll returns the same EffectHandle as the direct helper. It uses IntersectionObserver internally and disconnects after the first trigger, so the animation plays once per page load.

Note: The demo above plays when it scrolls into view — the same behaviour as createDecryptEffectOnScroll on a real page. Use the Replay button to re-run it.
